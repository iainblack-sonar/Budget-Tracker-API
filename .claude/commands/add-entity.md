Scaffold a new JPA entity for the Budgetly persistence layer. Follow these steps in order:

**Input required:** The user should provide the entity name and its fields.

## Steps

1. **Create the entity in `budgetly-persistence`** (`src/main/java/com/budgetly/persistence/entity/`)
   - Use `@Entity`, `@Table`, `@Id`, `@GeneratedValue(strategy = GenerationType.IDENTITY)`
   - Use Lombok: `@Getter`, `@Setter`, `@NoArgsConstructor`, `@AllArgsConstructor`, `@Builder`
   - Add `@Column` annotations with `nullable`, `unique`, `length` as appropriate
   - Include `createdAt` field with `@PrePersist` lifecycle callback
   - Include `updatedAt` if the entity is mutable, with `@PreUpdate`
   - If the entity belongs to a user, include `userId` (not a JPA relationship — keep it simple)
   - Create any needed enum types in the same package

2. **Create the repository interface** (`src/main/java/com/budgetly/persistence/repository/`)
   - Extend `JpaRepository<EntityName, Long>`
   - Add `@Repository`
   - Add `findAllByUserId(Long userId)` if the entity is user-scoped
   - Add `findByIdAndUserId(Long id, Long userId)` if the entity is user-scoped

3. **Create a Flyway migration script** (`budgetly-persistence/src/main/resources/db/migration/`)
   - Name it `V{next_number}__create_{table_name}_table.sql`
   - Check existing migrations to determine the next version number
   - Use `BIGINT GENERATED BY DEFAULT AS IDENTITY` for primary keys
   - Add foreign key constraints where appropriate
   - Add indexes on frequently queried columns (e.g., `user_id`)

4. **Create corresponding DTOs in `budgetly-api`** (`src/main/java/com/budgetly/api/dto/`)
   - Create request record(s) with validation annotations
   - Create response record with `static from(Entity)` factory method
   - Use Java `record` types

## Architecture reminders
- Entities live ONLY in `budgetly-persistence`
- DTOs live ONLY in `budgetly-api`
- The entity and DTO should NOT reference each other directly (the `from()` method on the DTO takes the entity as a parameter)
